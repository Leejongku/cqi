■ 현상
  - findLcqiPrevSectList.do (POST) 등 @ResponseBody 컨트롤러 호출 시,
    데이터가 없거나 정상 응답 후 SecurityEnforcer.postHandle(SecurityEnforcer.java:173) 에서
    java.lang.NullPointerException 발생.

■ 원인
  - @ResponseBody 메서드는 뷰 이름을 반환하지 않으므로, HandlerInterceptor.postHandle() 에
    전달되는 ModelAndView 가 null 일 수 있음.
  - SecurityEnforcer.postHandle() 173라인 근처에서 modelAndView 를 null 체크 없이 사용하여 NPE 발생.

■ 수정 위치 (kr.dku.base 프로젝트)
  - 클래스: kr.dku.base.security.access.SecurityEnforcer
  - 메서드: postHandle(Request request, Response response, Object handler, ModelAndView modelAndView)

■ 수정 방법
  - postHandle() 메서드 맨 앞에 아래 null 체크 추가:

    public void postHandle(HttpServletRequest request, HttpServletResponse response,
            Object handler, ModelAndView modelAndView) throws Exception {
        if (modelAndView == null) {
            return;  // @ResponseBody 등 뷰 미사용 요청
        }
        // ... 기존 173라인 부터 로직 ...
    }

  - 또는 173라인에서 modelAndView를 사용하기 직전에만 null 체크를 넣어도 됩니다.
    (해당 라인 코드에 따라 if (modelAndView != null) { ... } 로 감싸기)

■ 참고
  - Spring HandlerInterceptor 문서: postHandle의 modelAndView는 @ResponseBody 처리 시 null일 수 있음.
  - 교양 CQI 화면의 findLcqiMaster.do, findLcqiSectList.do, findLcqiPrevSectList.do, saveLcqiSect.do
    모두 @ResponseBody 이므로 동일 이슈 가능.
